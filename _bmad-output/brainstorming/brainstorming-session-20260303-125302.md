### Breakthrough Insights from Question Storming

**A. The App Is a Confidence Engine** — Not a logistics tool. It gives trip leaders the ability to SEE that things are on track, replacing invisible mental load with visible status.

**B. Progressive Disclosure Is the Core UX** — One link, one page, layers of depth. Casual friend sees date/location. Hardcore planner sees maps, rapids, meal plans. Trip leader curates what's available.

**C. The Leader Is a Publisher** — Not a shared doc coordinator. They publish a trip. Crew members are readers with different appetites. Some get contributor access for specific things.

**D. Per-Person Readiness Dashboard** — Trip readiness isn't one bar. It's per-person, and the leader can see everyone's state at a glance.

**E. Help Is Optional and Leader-Controlled** — Delegate sections (meals), but the leader controls the frame. Offering help ≠ requiring it.

**F. Post-Trip Brain Dump → Structured Learning** — Voice-to-text ramble → system parses into gear repairs, location intel, meal notes, action items. Every trip makes the next one better.

**G. Cost Ledger with Auto-Settlement** — Log receipts throughout, app calculates split, generates "who owes who" with payment links.

**H. Trip Templates from Past Trips** — Clone a Deschutes run, adjust dates/crew/meals, keep everything else.

**I. Product Boundary** — Day hikes = people meet up (not this app). Trips = people plan and live together (this app).

**J. Gear Closet Is Optional Power Layer** — Core product is Trip Command Center. Gear Closet enriches but never gates the experience.

## Technique 3: Morphological Analysis — Results

### Parameter 1: Trip Types
All trip types share the same module structure. Difference is depth, not presence.
- Multi-day raft trip (extreme gear, high coordination)
- Backcountry ski/hut trip (high gear, safety-critical)
- Backpacking trip (medium-high gear, medium coordination)
- Car camping (medium gear, medium coordination)
- Bike touring (medium gear, daily destinations)
- Ski weekend / resort (low gear, low coordination)
- General group travel (low gear, medium coordination)

### Parameter 2: Feature Modules (All Universal, Scale by Depth)
Trip Overview, Crew Roster, Itinerary, Meal Plan, Gear/Packing, Task Board, Resource Library, Cost Ledger, Reminders, Readiness Dashboard, Post-Trip Record, Participant Profiles, Documents/Permits, Emergency Info, Gear Closet (optional persistent layer)

### Parameter 3: User Roles
- **Trip Leader** — full ops dashboard, creates/manages everything
- **Co-Leader** — full or scoped access granted by leader
- **Contributor** — edit specific assigned sections (e.g. meals)
- **Participant** — view trip, mark readiness, update bring list, log expenses
- **Viewer** — read-only access

### Parameter 4: Trip Lifecycle
CREATE → PLAN → PREP → ACTIVE → POST-TRIP
- Leader experience is continuous across all phases
- Participant enters at PREP (gets the link), exits at POST-TRIP (costs settled)
- Trip closed out when funds settled, post-mortem done, maintenance addressed

### Parameter 5: The Universal Trip Object (Data Model)
```
TRIP = {
  overview:     { dates, route, description, type, map_link }
  crew:         [ { person, role, readiness, bring_list, expenses } ]
  itinerary:    [ { day, plan, location, notes } ]
  meals:        [ { day, meal, menu, cook, grocery_list } ]
  tasks:        [ { task, owner, status } ]
  resources:    [ { name, link_or_file, type } ]
  reminders:    [ { trigger_time, audience, message } ]
  costs:        [ { item, amount, paid_by, split_among, receipt } ]
  emergency:    { contacts, hospitals, evacuation_notes }
  post_trip:    { brain_dump, gear_notes, location_intel, learnings }
  gear_closet:  → optional link to persistent inventory
}
```

### Three App Entry Points
1. **"I'm leading a trip"** → Trip leader creating a new trip
2. **"I got invited"** → Participant via shared link
3. **"I want to organize my gear"** → Gear head using closet independently

### Key Decisions Confirmed
- **Gear Closet auto-created on registration** — populates passively as user packs for trips, or actively if they want to add items. Never required.
- **Wishlist section** in Gear Closet — create items with links to purchase
- **Pre-trip PDF** — leader-generated document, curated content (meals, maps, contacts, itinerary). Downloadable. Can be printed and taped to dry box.
- **No offline mode needed** — PDF solves the field access problem
- **Trip doesn't auto-archive** — leader closes it out after funds settled, post-mortem done, gear maintenance addressed
- **Gear deduplication** — participants can tag items on the trip gear list as "I have one" or add notes, so the group can see who has extras and avoid redundancy
- **Not a social network** — trips are private, invite-only
- **Not a route planner** — links out to Google Maps, CalTopo, etc.
- **Not a messaging app** — communication stays in existing channels, trip page is the reference
- **Not a booking tool** — tracks that bookings happened, doesn't make them
- **USA domestic only** for initial version
- **Day hikes are out of scope** — product boundary is "trips where people plan and live together"
- **Participants can view without heavy engagement** — the app meets people where they are

### Product Personality
> "Intentional and planned, but fun and loose and ready to dive into."
> For gear heads and trip leaders — a path that often intersects.
> The app is a confidence engine that turns invisible mental load into visible, shared status.

**Theme 4: Trip Lifecycle Management**

- Post-trip brain dump (voice-to-text → structured data → action items)
- Trip close-out is deliberate: funds settled, post-mortem done, gear maintenance noted
- Past trips become templates for future trips

**Theme 5: Financial Coordination**
- Cost ledger woven INTO trip planning
- Log expenses + receipts throughout, tagged to categories
- Auto-calculated "who owes who" with payment links

**Theme 6: Gear Closet (Independent + Optional Power Layer)**
- Auto-created on registration, passively populates from trip participation
- Independent value for gear heads
- Wishlist section with purchase links
- Condition tracking and maintenance flags
- Cross-trip intelligence and gear deduplication

**Theme 7: Resource & Knowledge Library**
- Trip-specific resource attachments
- Growing library that persists across trips

**Theme 8: Universal Architecture**
- Single trip data model scales by depth, not by module on/off
- Three entry points: leader, participant, gear head
- Product boundary: trips where people plan and live together

- Not a social network, route planner, messaging app, or booking tool

### Creative Facilitation Narrative
D came into this session with a working gear manifest prototype and a vision for "a trip leader app." Through Question Storming, we discovered the product isn't about logistics at all — it's about replacing the invisible mental load that burns out the person who always plans everything. The self-doubt, the 2am worry, the "did I forget something" — all of that comes from carrying an unverifiable system in your head. The app's job is to make the invisible visible so the leader can relax and actually enjoy the trip they planned.

Cross-Pollination revealed that wedding planners, restaurant kitchens, and project management tools have already solved most of these coordination problems — the outdoor recreation world just hasn't adopted them yet. The kitchen's "mise en place" became the pre-launch checklist. The wedding's guest page became the shareable trip link. Basecamp's calm structure became the design philosophy.

Morphological Analysis proved that the app doesn't need different modes for different trip types — one flexible data model scales by depth, not by presence. A raft trip and a ski weekend use the same structure, just at different levels of detail.

The strongest insight of the session: the Gear Closet — which started as the entire product idea — turned out to be an optional power layer. The real product is the Trip Command Center. And the Gear Closet becomes an acquisition funnel: gear heads discover trip planning, trip leaders discover gear management.
